\# 요구사항 명세서



1\. 프로젝트 개요



\- 프로젝트명 : 할 일 관리 웹앱 (Todo Manager)

\- 목적 : 개인 및 팀의 할 일을 효율적으로 관리하는 웹 애플리케이션 개발

\- 개발 방식 : Claude AI를 활용한 바이브코딩

\- 적용 프로세스 : 소프트웨어 공학 요구분석 방법론 적용



2\. 이해관계자 (Stakeholder)

\- 사용자 : 할 일을 등록하고 관리하는 일반 사용자

\- 개발자 : 시스템을 구축하고 유지보수하는 개발자



3\. 액터 (Actor)

\- 사용자 : 시스템과 직접 상호작용하는 주체. 할 일 등록/수정/삭제/조회 수행



4\. 유스케이스 목록

UC-01 : 할 일 추가 | 사용자 | 높음 |

UC-02 : 할 일 수정 | 사용자 | 높음 |

UC-03 : 할 일 삭제 | 사용자 | 높음 |

UC-04 : 완료 처리 | 사용자 | 높음 |

UC-05 : 우선순위 설정 | 사용자 | 중간 |

UC-06 : 마감일 설정 | 사용자 | 중간 |

UC-07 : 상태별 필터링 | 사용자 | 중간 |



5\. 유스케이스 관계

\- 포함 관계 (include) : UC-01 할 일 추가 → UC-05 우선순위 설정 포함

\- 포함 관계 (include) : UC-01 할 일 추가 → UC-06 마감일 설정 포함

\- 확장 관계 (extend) : UC-04 완료 처리 → UC-07 필터링에서 완료 항목 조회로 확장



6\. 기능 요구사항

FR-01 - 할 일 추가 | 제목, 우선순위, 마감일을 입력하여 할 일 등록 |

FR-02 - 할 일 수정 | 등록된 할 일의 내용을 수정 |

FR-03 - 할 일 삭제 | 등록된 할 일을 삭제 |

FR-04 - 완료 체크 | 할 일의 완료 여부를 체크/해제 |

FR-05 - 우선순위 설정 | 높음 / 중간 / 낮음 3단계로 우선순위 설정 |

FR-06 - 마감일 설정 | 날짜 선택으로 마감일 지정 |

FR-07 - 상태 필터링 | 전체 / 진행중 / 완료 기준으로 목록 필터링 |



7\. 비기능 요구사항

NFR-01 : 사용성 | 처음 사용자도 직관적으로 사용 가능해야 함 |

NFR-02 : 반응성 | 버튼 클릭 후 1초 이내 결과 표시 |

NFR-03 : 호환성 | Chrome, Safari 등 주요 브라우저에서 동작 |

NFR-04 : 유지보수성 | SOLID 원칙 준수로 코드 수정 용이 |



8\. 제약사항

\- 별도 서버 없이 브라우저에서 단독 실행 가능한 구조

\- 데이터는 브라우저 로컬 스토리지에 저장

\- 외부 라이브러리 최소화
