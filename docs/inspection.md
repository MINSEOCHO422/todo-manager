\- 대상 파일: src/index.html





2\. 코드 스멜 체크 (Chap 09 리팩토링 기준)



1\. 중복된 코드 → 발견 없음

2\. 긴 메서드 → 발견 없음

3\. 큰 클래스 → 발견 없음

4\. 긴 파라미터 리스트 → 발견 없음





3\. SOLID 원칙 준수 확인 (Chap 06 설계원리)



1\. SRP (단일 책임) → O, Model/View/Controller 역할 분리

2\. OCP (개방 폐쇄) → O, 필터 기능 확장 가능한 구조

3\. LSP (리스코프) → O, 상속 구조 없음 (해당 없음)

4\. ISP (인터페이스 분리) → O, 각 객체가 필요한 메서드만 보유

5\. DIP (의존관계 역전) → O, Controller가 Model/View에 직접 의존 최소화





4\. UI 사용성 체크 (Chap 08 UI설계)



1\. 단순하고 자연스럽게 → O, 직관적인 레이아웃

2\. 즉각적인 피드백 → O, 추가/삭제 즉시 화면 반영

3\. 오류 회복 용이 → O, 삭제 전 confirm 확인창

4\. 일관성 유지 → O, 동일한 버튼 스타일 사용

5\. 제약 제공 → O, 빈 입력 시 alert 표시





5\. 발견된 결함 및 개선 사항
