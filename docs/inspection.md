3-2. 코드 안전성 증명 (정적 분석)

- toggle(): id 미발견 시 조기 반환 -> null 참조 오류 없음
- update(): id 미발견 시 조기 반환 -> null 참조 오류 없음
- addTodo(): 빈 입력 및 과거 날짜 유효성 검사 -> 오류 입력 방지


4. 블랙박스 테스트 케이스 (Chap 10.2)

4-1. 동등 분할 기법 적용

할일 제목 입력
- 유효 클래스: 1자 이상 텍스트 입력
- 무효 클래스: 빈 입력 (공백 포함)
- 예상 결과: 빈 입력 시 alert + 빨간 테두리 표시

마감일 입력
- 유효 클래스: 오늘 이후 날짜
- 무효 클래스: 오늘 이전 날짜
- 예상 결과: 과거 날짜 입력 시 빨간 텍스트 표시 후 추가 차단

우선순위 입력
- 유효 클래스: 높음, 중간, 낮음
- 무효 클래스: 없음 (탭 버튼으로만 선택 가능)
- 예상 결과: 기본값 중간으로 설정

4-2. 경곗값 분석 적용

마감일 경곗값
- 어제 날짜: 추가 차단 (무효)
- 오늘 날짜: 추가 가능 (유효)
- 내일 날짜: 추가 가능 (유효)


5. 화이트박스 테스트 케이스 (Chap 10.3)

5-1. 분기 커버리지

addTodo() 함수
- TC-01: 빈 제목 입력 -> alert 표시, 추가 안됨 (False 분기)
- TC-02: 정상 제목 입력, 마감일 없음 -> 추가 성공 (True 분기)
- TC-03: 정상 제목 입력, 과거 날짜 -> 빨간 텍스트, 추가 안됨
- TC-04: 정상 제목 입력, 오늘 이후 날짜 -> 추가 성공

filter() 함수
- TC-05: type = 'all' -> 전체 목록 반환
- TC-06: type = 'active' -> 미완료 항목만 반환
- TC-07: type = 'completed' -> 완료 항목만 반환
- TC-08: type = 'today' -> 오늘 마감 항목만 반환
- TC-09: type = 'high' -> 높음 우선순위 항목만 반환

isOverdue() 함수
- TC-10: dueDate = null -> false 반환
- TC-11: dueDate = 과거 날짜 -> true 반환
- TC-12: dueDate = 오늘 날짜 -> false 반환


6. SOLID 원칙 준수 확인 (Chap 06)

1. SRP (단일 책임) -> O, Model/View/Controller 역할 분리 유지
2. OCP (개방 폐쇄) -> O, filter() 메서드 타입 추가만으로 확장 가능
3. LSP (리스코프) -> O, 상속 구조 없음 (해당 없음)
4. ISP (인터페이스 분리) -> O, 각 객체가 필요한 메서드만 보유
5. DIP (의존관계 역전) -> O, Controller가 Model/View에 직접 의존 최소화
