2. 주요 변경 이력

3. SOLID 원칙 준수 확인 (Chap 06 설계원리)

1. SRP (단일 책임) -> O, Model/View/Controller 역할 분리 유지
2. OCP (개방 폐쇄) -> O, 필터/정렬 기능 확장 가능한 구조 유지
3. LSP (리스코프) -> O, 상속 구조 없음 (해당 없음)
4. ISP (인터페이스 분리) -> O, 각 객체가 필요한 메서드만 보유
5. DIP (의존관계 역전) -> O, Controller가 Model/View에 직접 의존 최소화


4. UI 사용성 체크 (Chap 08 UI설계)

1. 단순하고 자연스럽게 -> O, Notion+Apple 스타일로 직관적 레이아웃
2. 즉각적인 피드백 -> O, 추가/삭제/수정 즉시 화면 반영
3. 오류 회복 용이 -> O, 삭제 전 confirm, 빈 입력 시 빨간 테두리
4. 일관성 유지 -> O, 동일한 디자인 시스템 적용
5. 단축 명령 제공 -> O, 엔터키로 추가
6. 접근성 -> O, 다크모드, 모바일 반응형 지원
7. 인식하기 쉽게 -> O, 우선순위 뱃지 텍스트로 명확히 표시


5. 코딩 표준 체크 (Chap 09 코딩 표준)

1. 명명 규칙 -> O, 카멜케이스 일관되게 적용
2. 주석 -> O, 섹션별 주석 추가
3. 들여쓰기 -> O, 일관된 들여쓰기 적용
4. 오류 처리 -> O, 빈 입력/날짜 유효성 검사 적용
5. 모듈화 -> O, Model/View/Controller 분리


6. 발견된 결함 및 개선 사항

1. D-01, 카카오톡 인앱 브라우저 미지원, 심각도: 낮음 -> 사파리로 열기 안내로 대체
2. D-02, 아이폰 과거 날짜 선택 UI 제한 미적용, 심각도: 낮음 -> JS 유효성 검사로 대체
3. D-03, 멀티 디바이스 데이터 동기화 미지원, 심각도: 중간 -> v2 Supabase 연동 예정
